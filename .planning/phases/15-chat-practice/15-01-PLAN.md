<task type="auto" tdd="true">
  <name>Task 1: Create Zustand chat store with sendMessage and sendHint</name>
  <files>lib/store/chat.ts, __tests__/lib/chat-store.test.ts</files>
  <behavior>
    - sendMessage adds user message, creates assistant message with streaming:true, accumulates deltas, sets streaming:false on completion, attaches citations from RAG retrieval
    - sendMessage calls buildSystemPrompt with persona, lessonContext, and retrieved RAG chunks
    - sendMessage caps conversation history at last 6 messages (3 exchanges) before passing to engine
    - sendHint with error builds error-explanation prompt ("Explain what went wrong... Do NOT write corrected code")
    - sendHint without error builds Socratic hint prompt ("Give a Socratic hint... Do NOT reveal the answer")
    - openPanel/closePanel toggle isOpen state
    - setLessonContext updates the stored lesson context
    - Multiple sendMessage calls accumulate messages (multi-turn history preserved)
  </behavior>
  <action>
    Create `lib/store/chat.ts` as a Zustand store (non-persisted). Follow the exact pattern from `lib/store/progress.ts` for store creation.

    Export types:
    - `ChatMessage = { role: 'user' | 'assistant'; content: string; streaming: boolean; citations: RetrievedChunk[] }`
    - `useChatStore` (the Zustand hook)

    Store state:
    - `messages: ChatMessage[]` (starts empty)
    - `isOpen: boolean` (starts false)
    - `lessonContext: LessonContext | null` (starts null)

    Store actions:
    - `setLessonContext(ctx)` — updates lessonContext
    - `openPanel()` — sets isOpen: true
    - `closePanel()` — sets isOpen: false
    - `sendMessage(userText, getEngine, retrieveContext, persona)` — full orchestration:
      1. Push user message `{ role: 'user', content: userText, streaming: false, citations: [] }`
      2. Push empty assistant message `{ role: 'assistant', content: '', streaming: true, citations: [] }`
      3. Call `const engine = await getEngine()`
      4. Call `const ragChunks = await retrieveContext(userText, engine, 3)`
      5. Call `buildSystemPrompt(persona, get().lessonContext!, ragChunks)`
      6. Slice conversation history to last 6 messages (before the new user message)
      7. Call `engine.chat.completions.create({ messages: [...], stream: true, temperature: 0.7, max_tokens: 512 })`
      8. `for await` delta loop: accumulate `delta.content` into last message via `set()`
      9. In `finally` block: set `streaming: false` and attach `citations: ragChunks` to last message
    - `sendHint(code, error, exerciseDescription, getEngine, retrieveContext, persona)` — same flow but builds hint prompt instead:
      - If error is non-null: error explanation prompt (explain error in plain English, ONE guiding question, do NOT write corrected code)
      - If error is null: Socratic hint prompt (guiding question or partial clue, do NOT reveal the answer)
      - Uses `max_tokens: 256` (shorter responses for hints)
    - `clearMessages()` — resets messages to empty array

    CRITICAL anti-patterns to avoid:
    - Do NOT persist messages to localStorage. No Zustand `persist` middleware.
    - Do NOT call getEngine() or retrieveContext() outside of sendMessage/sendHint.
    - `streaming: false` MUST be set in a `finally` block after the `for await` loop, never before the generator is exhausted.

    Create `__tests__/lib/chat-store.test.ts`:
    - Mock `buildSystemPrompt` from `@/lib/build-system-prompt`
    - Create a mock engine with `chat.completions.create` returning an async generator of deltas
    - Create a mock `retrieveContext` returning sample `RetrievedChunk[]`
    - Test all behaviors listed above
    - Use `act()` from `@testing-library/react` for state updates
    - Reset store between tests using `useChatStore.setState({ messages: [], isOpen: false, lessonContext: null })`
  </action>
  <verify>
    <automated>pnpm test -- chat-store</automated>
  </verify>
  <done>Chat store passes all tests: sendMessage accumulates streaming tokens, attaches citations, caps history at 6 messages; sendHint builds correct Socratic/error prompts; openPanel/closePanel toggle state.</done>
</task>
