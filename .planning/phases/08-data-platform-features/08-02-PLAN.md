PART A — Worker: Extend `public/workers/pyodide.worker.mjs`

The approach is to wrap the user's code in a Python harness that (a) mounts dataset files into the Pyodide FS, (b) executes the user code, and (c) inspects the last expression's value to decide how to render it.

1. **Dataset mounting** — At the top of the message handler, after Pyodide is ready, mount all bundled
   datasets into the Pyodide filesystem. Fetch them from the worker's own origin using `fetch`:

   ```js
   async function ensureDatasetsMounted(pyodide) {
     if (pyodide._datasetsMounted) return
     pyodide._datasetsMounted = true
     try {
       pyodide.FS.mkdir('/data')
     } catch (_) { /* already exists */ }
     const datasets = ['students.csv']
     for (const file of datasets) {
       const res = await fetch(`/data/${file}`)
       const text = await res.text()
       pyodide.FS.writeFile(`/data/${file}`, text)
     }
   }
   ```

   Call `await ensureDatasetsMounted(pyodide)` after Pyodide loads and before running code.
   Files become accessible as `pd.read_csv('data/students.csv')` (Pyodide's CWD is `/`).

2. **Output detection harness** — Wrap the user code execution:

   ```js
   const harness = `
   import sys as _sys
   _user_code = """${code.replace(/\\/g, '\\\\').replace(/"""/g, '\\"\\"\\"')}"""
   _result = None
   try:
       import ast as _ast
       _tree = _ast.parse(_user_code, mode='exec')
       if _tree.body and isinstance(_tree.body[-1], _ast.Expr):
           _expr = _ast.Expression(body=_tree.body[-1].value)
           _stmts = _ast.Module(body=_tree.body[:-1], type_ignores=[])
           exec(compile(_stmts, '<string>', 'exec'), globals())
           _result = eval(compile(_expr, '<string>', 'eval'), globals())
       else:
           exec(compile(_tree, '<string>', 'exec'), globals())
   except Exception as _e:
       raise _e
   `

   await pyodide.runPythonAsync(harness)
   const result = pyodide.globals.get('_result')
   ```

   Then inspect `result`:
   - `result` is null/None → no extra output line
   - `result` is a pandas DataFrame (check `result.constructor.name === 'DataFrame'` via JS, or use
     `pyodide.runPythonAsync("import pandas as pd; isinstance(_result, pd.DataFrame)")`) →
     push `{ type: 'html', line: await pyodide.runPythonAsync("_result.to_html(classes='df-table', border=0, max_rows=50)") }`
   - `result` is a pandas Series or Index → push `{ type: 'stdout', line: await pyodide.runPythonAsync("str(_result)") }`
   - Any other value → push `{ type: 'stdout', line: await pyodide.runPythonAsync("repr(_result)") }`

   IMPORTANT: Only do the pandas type check if pandas was already installed (check `pandasReady !== null`
   to avoid importing pandas just to do a type check when the user hasn't used pandas). For non-pandas
   code, use a simpler path: `result !== null && result !== undefined` → push repr as stdout.

   CAUTION on string escaping in the harness: the `${code}` interpolation must escape backslashes and
   triple-quotes in user code to avoid breaking the harness string. Use the replace chain shown above.
   A cleaner alternative is to use `pyodide.globals.set('_user_src', code)` before the harness, then
   reference `_user_src` as a Python variable in the harness instead of string interpolation — this
   eliminates all escaping issues. Prefer this approach.

3. **Error handling**: keep the existing try/catch in the message handler. The harness itself re-raises
   Python exceptions, so Python errors surface as JS `err.message` as before.

   the hook passes it through unchanged.

2. **Update `components/code-runner/output-panel.tsx`** — add HTML rendering branch:

   The OutputPanel currently maps over output lines and renders each as a `<pre>`. Extend the map to
   add a third case for `type === 'html'`:

   ```tsx
   {item.type === 'html' ? (
     <div
       key={index}
       className="df-output overflow-x-auto"
       dangerouslySetInnerHTML={{ __html: item.line }}
     />
   ) : (
     <pre
       key={index}
       className={item.type === 'stderr'
         ? 'text-red-500 dark:text-red-400 whitespace-pre-wrap break-all m-0'
         : 'whitespace-pre-wrap break-all m-0'
       }
     >
       {item.line}
     </pre>
   )}
   ```

   Additionally, add scoped table styles via a `<style>` tag rendered once when any html output
   exists. Place it before the output map:

   ```tsx
   {output.some(o => o.type === 'html') && (
     <style>{`
       .df-output table { border-collapse: collapse; font-size: 0.8rem; width: 100%; }
       .df-output th { background: var(--color-background-muted, #f3f4f6); font-weight: 600; text-align: left; padding: 4px 8px; border: 1px solid var(--color-border, #e5e7eb); }
       .df-output td { padding: 3px 8px; border: 1px solid var(--color-border, #e5e7eb); }
       .df-output tr:nth-child(even) { background: var(--color-background-subtle, #f9fafb); }
     `}</style>
   )}
   ```

   This approach uses CSS custom properties already defined in the app's design system, so the table
   respects both light and dark themes without adding a Tailwind dependency on dynamic HTML.

3. **Update `__tests__/components/code-runner.test.tsx`** — add OutputPanel tests for html type:

   ```typescript
   it('renders html output as dangerouslySetInnerHTML table', async () => {
     const { OutputPanel } = await import('@/components/code-runner/output-panel')
     const htmlTable = '<table><thead><tr><th>name</th></tr></thead><tbody><tr><td>Alice</td></tr></tbody></table>'
     const { container } = render(
       React.createElement(OutputPanel, {
         output: [{ type: 'html', line: htmlTable }],
         error: null,
       })
     )
     expect(container.querySelector('.df-output')).not.toBeNull()
     expect(container.querySelector('table')).not.toBeNull()
     expect(container.querySelector('th')?.textContent).toBe('name')
   })

   it('does not escape html output as text', async () => {
     const { OutputPanel } = await import('@/components/code-runner/output-panel')
     const { queryByText } = render(
       React.createElement(OutputPanel, {
         output: [{ type: 'html', line: '<table></table>' }],
         error: null,
       })
     )
     expect(queryByText('<table></table>')).toBeNull()
   })
   ```

SECURITY NOTE: The HTML injected via `dangerouslySetInnerHTML` is exclusively generated by
`pandas.DataFrame.to_html()` running inside the Pyodide sandbox. It is not user-controlled HTML
(users write Python, not HTML). The risk profile is equivalent to rendering sanitized content.
No additional sanitization is required for this use case.
