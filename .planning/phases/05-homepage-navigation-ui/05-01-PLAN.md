       - `<header>` with `sticky top-0 z-40 h-16` (remove static `border-b bg-background` — these are now conditional via HeaderClient)

    4. Create `components/header-client.tsx` as a 'use client' component:
       - Import `usePathname` from next/navigation, `useScrolled` from hook
       - `useScrolled(10)` for scroll detection
       - When `scrolled` is true, apply `backdrop-blur-sm bg-background/80 border-b` on a wrapper div that spans the full header area. Use a `data-scrolled` attribute pattern: render a div that wraps the header inner content and conditionally adds border/blur classes.
       - When `pathname === '/'`, render a `<Button asChild size="sm" className="rounded-full">` wrapping `<Link href="/courses/python">Start Learning</Link>`
       - Always render `<SearchDialog />` and `<ThemeToggle />`
       - The scroll border effect: since HeaderClient is inside the header, use a useEffect to set `data-scrolled` attribute on the parent `<header>` element via `document.querySelector('header')` or pass the scrolled state to a className on the header's inner container. Simplest approach: HeaderClient renders inside the header's flex container, and applies border/blur to the entire header by adding/removing a class on document.body or using a CSS approach. The cleanest: make the `<header>` element part of HeaderClient by wrapping the entire header in HeaderClient. But research recommends keeping header server-rendered. Best compromise: HeaderClient renders a full-width absolute-positioned backdrop div behind the header content that shows border-b + blur when scrolled.

       Actually, simplest correct approach: Make `SiteHeader` render the `<header>` tag with base classes, and have `HeaderClient` use a `useEffect` to toggle a `data-scrolled` attribute on the closest `<header>` ancestor. Add CSS in globals.css: `header[data-scrolled] { @apply backdrop-blur-sm bg-background/80 border-b border-border; }`. Without `data-scrolled`, header has no border-b or backdrop-blur (just transparent/bg-background).

    5. Create `__tests__/components/site-header.test.tsx`: Test that SiteHeader renders the JustLearn link with href="/". Test HeaderClient with mocked usePathname: when "/" shows Start Learning button, when "/courses/python" does not.
