    2. Create hooks/use-rag.ts with module-level singleton pattern (mirrors use-ai-engine.ts exactly):
       - Module-level variables: `chunksCache: RagChunk[] | null`, `indexPromise: Promise<void> | null`, `courseChunksCache: Map<string, RagChunk[]>`
       - Export type `RAGStatus = 'idle' | 'loading' | 'ready' | 'error'`

    3. **Use plain cosine similarity search instead of mememo.** The research identified mememo's API surface as uncertain (Open Question 1) and recommended cosine similarity as an acceptable fallback at <1,500 chunks. Since the corpus is ~1,100-2,200 chunks and O(n) scan is fast enough, start with the dependency-free approach:
       ```typescript
       function cosineSimilarity(a: Float32Array, b: Float32Array): number {
         let dot = 0, normA = 0, normB = 0
         for (let i = 0; i < a.length; i++) {
           dot += a[i] * b[i]
           normA += a[i] * a[i]
           normB += b[i] * b[i]
         }
         return dot / (Math.sqrt(normA) * Math.sqrt(normB))
       }
       ```
       This eliminates the mememo dependency risk entirely. If mememo is needed later for scale, it can replace the cosine scan in this one file.

    4. buildIndex() implementation:
       - If chunksCache exists, return immediately (singleton)
       - If indexPromise exists, return it (dedup concurrent callers)
       - Fetch `/data/rag-chunks.json`, parse as RagChunk[]
       - Store in chunksCache
       - Pre-filter by courseSlug and store in courseChunksCache map
       - Convert each chunk's embedding number[] to Float32Array for fast cosine math
       - Set status 'ready'
       - On error: reset indexPromise to null, set status 'error', re-throw

    5. retrieveContext(query, engine, k=3) implementation:
       - Call buildIndex() to ensure data is loaded
       - Call `engine.embeddings.create({ input: query })` to get query vector (reuses already-loaded WebLLM engine — zero additional model download)
       - Convert to Float32Array
       - Compute cosine similarity against all course-filtered chunks
       - Sort descending, take top-k
       - Return as `RetrievedChunk[]` (text, heading, lessonTitle)

    6. For tests, mock `global.fetch` to return a fake RagChunk[] array with known embeddings. Mock the engine's `embeddings.create()` to return a known query vector. This tests the full flow without real models. Reset module-level singletons between tests by exporting a `_resetForTesting()` function (prefixed with underscore to signal internal use).

    IMPORTANT: Do NOT install or import mememo. Use plain cosine similarity. The RESEARCH.md explicitly documents this as an acceptable approach at this corpus size, and mememo's API is flagged as uncertain.
