  <behavior>
    - PracticeBlock renders header with Code2 icon and "Try it yourself" label
    - PracticeBlock renders prompt text
    - PracticeBlock renders CodeRunner with initialCode prop
    - When hint is provided, "Show hint" button appears; clicking toggles hint visibility
    - When solution is provided, "Show solution" button appears; clicking toggles solution visibility
    - When hint is not provided, no hint button renders
    - When solution is not provided, no solution button renders
  </behavior>
  <action>
1. Create `components/practice-block.tsx` as a Client Component (`'use client'`):
   - Props: `{ prompt: string, initialCode?: string, hint?: string, solution?: string }`
   - Default `initialCode` to `'# Write your code here\n'`
   - State: `hintOpen` and `solutionOpen` booleans, both default false
   - Layout per design spec section 12.7:
     - Outer: `not-prose my-8 overflow-hidden rounded-lg border border-[var(--color-border)] bg-[var(--color-background-subtle)]`
     - Header: 44px, flex, items-center, gap-2, border-b, bg `--color-background-muted`, px-4
       - Code2 icon from lucide-react, 18px, `--color-primary`
       - "Try it yourself" label, text-sm font-semibold
     - Prompt: px-4 py-3 text-sm
     - CodeRunner: wrapped in px-4 pb-4 div
     - Hint section (conditional on `hint` prop): border-t, px-4 py-3
       - Ghost button "Show hint" / "Hide hint"
       - Content: text-sm, `--color-foreground-muted`, mt-2
     - Solution section (conditional on `solution` prop): border-t, px-4 py-3
       - Ghost button "Show solution" / "Hide solution"
       - Content: `<pre>` with code styling, overflow-x-auto, rounded-md, bg `--color-code-bg`, p-3, text-sm

   IMPORTANT: CodeRunner is already dynamically imported with `ssr: false` internally — do NOT add another dynamic import wrapper. Just `import { CodeRunner } from '@/components/code-runner'` directly.

2. Create `__tests__/components/practice-block.test.tsx`:
   - Mock `@/components/code-runner` to return a simple div (avoids Pyodide/worker complexity in tests)
   - Test: header renders with "Try it yourself" text
   - Test: prompt text renders
   - Test: CodeRunner receives initialCode prop
   - Test: hint button not rendered when hint prop omitted
   - Test: hint button toggles hint visibility on click
   - Test: solution button not rendered when solution prop omitted
   - Test: solution button toggles solution visibility on click
   - Test: both hint and solution can be open simultaneously
