**lesson-01-what-is-data-engineering.md** — "What is Data Engineering?"
Duration: 30 minutes. Level: Intermediate.
Content (all prose, no code):
- Part 1: The data problem every company faces — raw data vs usable data, why pipelines exist
- Part 2: Data engineering defined — building and maintaining systems that collect, store, and process data at scale; contrast with data science (DE builds the roads, DS drives the cars) and data analytics (DE produces clean data, analysts consume it)
- Part 3: What data engineers actually do — design pipelines, build ETL/ELT workflows, manage data infrastructure, ensure data availability and quality
- Part 4: A day in the life — realistic workflow examples: debugging a failed pipeline, adding a new data source, optimizing a slow query
- Key Takeaways, Common Mistakes (confusing DE with data science, thinking DE is only for big companies), Next Lesson Preview: The Modern Data Stack

**lesson-02-the-modern-data-stack.md** — "The Modern Data Stack"
Duration: 40 minutes. Level: Intermediate.
Content:
- Part 1: What is a "stack"? — layered architecture overview
- Part 2: The four layers — Data Sources (APIs, databases, files, events), Ingestion & Orchestration (Airbyte, Fivetran, Airflow, dbt), Storage (data warehouses: Snowflake/BigQuery/Redshift, data lakes: S3/GCS), Serving & Analytics (BI tools, ML pipelines, APIs)
- Part 3: Open-source vs managed services — tradeoffs (cost, control, maintenance burden)
- Part 4: How the layers connect — data flow narrative from raw source event to analyst dashboard
- Use `<Info>` callout: "In this course we focus on the Python-level skills that are portable across any stack configuration."
- Key Takeaways, Common Mistakes (over-engineering early, choosing tools before understanding data), Next Lesson Preview

**lesson-03-python-for-data-engineering.md** — "Python for Data Engineering"
Duration: 35 minutes. Level: Intermediate.
Content:
- Part 1: Why Python dominates DE — readability, ecosystem breadth, community, hiring market
- Part 2: The core DE libraries — pandas (data manipulation), SQLAlchemy (database connections), requests (API ingestion), boto3 (AWS S3), pyarrow (Parquet/Arrow), great_expectations (data quality), Airflow (orchestration)
- Part 3: Python vs Scala/Java — when Python is enough (90% of cases), when JVM matters (massive-scale streaming)
- Part 4: What you'll build in this course — mental map of skills across all 10 sections, tying back to each library
- Use `<Tip>` callout: "You already know Python syntax from Course 1. This course teaches you to apply it to data problems — you won't need to learn a new language."
- Key Takeaways, Common Mistakes, Next Lesson Preview

**lesson-04-data-pipeline-concepts.md** — "Data Pipeline Concepts"
Duration: 40 minutes. Level: Intermediate.
Content:
- Part 1: What is a pipeline? — analogy: water pipeline (source → treatment → distribution), data pipeline (source → transform → destination)
- Part 2: Anatomy of a pipeline — Sources (databases, APIs, files, message queues), Extractors (connectors that read from sources), Transformers (business logic applied to raw data), Loaders (write to destination), Destinations (warehouse, database, file storage)
- Part 3: ETL vs ELT — Extract-Transform-Load (transform before storage, good for structured targets) vs Extract-Load-Transform (load raw first, transform in warehouse, good for modern cloud warehouses)
- Part 4: Pipeline reliability — idempotency (running twice produces same result), incremental loading vs full refresh, failure modes and recovery
- Use `<Warning>` callout: "Non-idempotent pipelines are the #1 source of duplicate data bugs. Always design for 'safe to re-run'."
- Key Takeaways, Common Mistakes (ignoring idempotency, skipping incremental design), Next Lesson Preview

**lesson-05-batch-vs-streaming.md** — "Batch Processing vs Streaming"
Duration: 40 minutes. Level: Intermediate.
Content:
- Part 1: Batch processing — process data in chunks at scheduled intervals (hourly, daily); classic paradigm; examples: nightly sales reports, end-of-day ETL jobs
- Part 2: Streaming (real-time) processing — process events as they arrive; examples: fraud detection, live dashboards, ride-sharing surge pricing; tools: Kafka, Flink, Spark Streaming
- Part 3: When to choose each — batch: latency tolerance > minutes, historical analysis, simpler infrastructure; streaming: latency requirement < seconds, event-driven systems, higher operational complexity
- Part 4: Micro-batching as a middle ground — process every 30 seconds; Spark Structured Streaming, AWS Kinesis Firehose
- Use `<Info>` callout: "90% of DE work is batch processing. Learn batch well before diving into streaming."
- Key Takeaways: 5 clear distinctions, Common Mistakes (streaming everything, ignoring cost), Next Lesson Preview

**lesson-06-data-storage-fundamentals.md** — "Data Storage Fundamentals"
Duration: 45 minutes. Level: Intermediate.
Content:
- Part 1: Relational databases (OLTP) — structured data, ACID transactions, row-oriented storage, PostgreSQL/MySQL; when to use (transactional systems, low-latency reads)
- Part 2: Data warehouses (OLAP) — columnar storage, analytical queries, denormalized schemas, Snowflake/BigQuery/Redshift; when to use (analytics, aggregation, BI)
- Part 3: Data lakes — raw storage of any format (CSV, JSON, Parquet, images), schema-on-read, S3/GCS/Azure Blob; when to use (keep raw data, ML feature stores)
- Part 4: The Lakehouse pattern — combines lake storage (cheap, flexible) with warehouse query performance (Delta Lake, Apache Iceberg, Databricks)
- Part 5: File formats matter — CSV (universal, no schema), JSON (nested, verbose), Parquet (columnar, compressed, efficient for analytics) — foreshadow Section 3
- Use `<Tip>` callout: "Choosing the wrong storage layer is an expensive mistake. Start with: do I need transactions (DB), analytics (warehouse), or raw archive (lake)?"
- Key Takeaways, Common Mistakes, Next Lesson Preview

**lesson-07-data-quality-overview.md** — "Introduction to Data Quality"
Duration: 35 minutes. Level: Intermediate.
Content:
- Part 1: Why data quality matters — garbage in, garbage out; business decisions made on bad data; regulatory and compliance risk; real examples of costly data quality failures
- Part 2: The 6 dimensions of data quality — Completeness (no missing critical fields), Accuracy (values are correct), Consistency (same definition across systems), Timeliness (data arrives when needed), Validity (values conform to expected format/range), Uniqueness (no unintended duplicates)
- Part 3: Where quality breaks down — at ingestion (source system issues), during transformation (logic errors, type mismatches), at joins (fan-out multiplying rows), over time (schema drift, upstream changes)
- Part 4: Quality as an ongoing practice — data contracts, monitoring, alerting, data lineage; preview of Section 8 (Data Quality & Testing)
- Use `<Warning>` callout: "Data quality is not a one-time fix. It requires monitoring infrastructure — a pipeline that passes today can silently break next week due to upstream schema changes."
- Key Takeaways, Common Mistakes (treating quality as a cleanup task, no monitoring), Next Lesson Preview

**lesson-08-de-roles-and-tools.md** — "Data Engineering Roles and the Tooling Landscape"
Duration: 35 minutes. Level: Intermediate.
Content:
- Part 1: The data team ecosystem — Data Engineer (builds pipelines, infra), Analytics Engineer (models and transforms data for BI, dbt), Data Analyst (queries and visualizes), Data Scientist (ML, statistics), MLOps Engineer (ML infrastructure); overlapping responsibilities
- Part 2: The typical DE toolbox — Orchestration (Apache Airflow, Prefect, Dagster), Ingestion (Fivetran, Airbyte, custom Python scripts), Transformation (dbt, pandas, Spark), Storage (Snowflake, BigQuery, S3), Quality (Great Expectations, dbt tests, Monte Carlo), Serving (REST APIs, BI tools)
- Part 3: What this course covers vs what it doesn't — this course uses Python directly (no Airflow, no dbt) to build foundational skills that transfer to any tool; explain why starting with raw pandas is better than starting with a framework
- Part 4: Your learning path from here — visual map of Sections 2-10 and what each section builds toward; how to use this course alongside real projects
- Use `<Info>` callout: "You don't need to master every tool before your first DE job. Employers care most about: SQL, Python/pandas, one cloud platform, and understanding of pipelines."
- Key Takeaways, Common Mistakes (tool obsession, ignoring fundamentals), final footer link back to README (no "Next Lesson" as this is last in section)
