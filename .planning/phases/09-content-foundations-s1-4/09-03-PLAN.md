**lesson-07-parquet-format.md** — "The Parquet Format"
Duration: 45 minutes. Level: Intermediate.
Content:
- Part 1: What is Parquet? — columnar binary format developed for Hadoop, now the standard for data lakes
- Part 2: Why columnar matters — analytical queries read only needed columns; compression is per-column (same data type compresses better); row formats (CSV) read entire rows for every query
- Part 3: `pd.read_parquet()` and `to_parquet()` — pyarrow or fastparquet as engine; `columns` parameter to read subset
- Part 4: Parquet vs CSV comparison — size (3-10x smaller), speed (5-10x faster for analytics), schema preserved (no dtype inference problems), not human-readable
- Part 5: Parquet in practice — standard format for S3 data lakes, BigQuery external tables, Spark
- Use `<Info>`: "When in doubt about file format for a new data pipeline, use Parquet. It's the industry default for data engineering work."
- Two PracticeBlocks:
  1. Load orders.csv and write it as Parquet with `to_parquet('data/orders.parquet', index=False)`. Then read it back and compare dtypes — notice that datetime and numeric types are preserved without re-parsing.
  2. Read only the 'customer' and 'unit_price' columns from the parquet file using the `columns` parameter.

**lesson-08-chunked-reading.md** — "Chunked Reading for Large Files"
Duration: 40 minutes. Level: Intermediate.
Content:
- Part 1: The memory problem — files larger than available RAM crash pandas with MemoryError; solution: process in chunks
- Part 2: `pd.read_csv(chunksize=N)` — returns a TextFileReader iterator; each iteration yields a DataFrame of N rows
- Part 3: Processing pattern — accumulate results from each chunk, then combine: compute a statistic per chunk, collect results, aggregate
- Part 4: When to use chunking vs other solutions — chunking (medium files, single machine), Dask (parallel pandas API), Spark (distributed)
- Part 5: Estimating chunk size — rule of thumb: chunk_size = available_memory_bytes / (row_size_bytes * 3) for safety margin
- Use `<Tip>`: "For a 1GB CSV on a 4GB RAM machine, use chunksize=100_000. Each chunk will be ~10-20MB, leaving plenty of headroom."
- Two PracticeBlocks:
