**lesson-01-groupby-basics.md** — "GroupBy: Splitting and Aggregating Data"
Course header: Data Transformation | 2 hours | Intermediate

Learning Objectives:
- Understand the split-apply-combine pattern
- Use groupby with single and multiple keys
- Apply built-in aggregation functions (sum, mean, count, max, min, std)
- Use .agg() with a dictionary for per-column aggregation

Part 1: The Split-Apply-Combine Pattern (30 min)
- Explain the mental model: split data into groups, apply a function to each group, combine results
- Show groupby with a single key: `df.groupby("region")["revenue"].sum()`
- Show groupby object properties: groups, ngroups, get_group()
- Runnable example: sales DataFrame with region/product/revenue columns, grouped by region

Part 2: Built-in Aggregation Functions (30 min)
- Show all common agg functions: sum, mean, count, size, max, min, std, var, median, nunique
- Show multi-column aggregation: `df.groupby("region")[["revenue", "quantity"]].sum()`
- Show multiple keys: `df.groupby(["region", "category"])["revenue"].sum()`
- Runnable example: compute revenue stats by region (total, average, count, max)

Part 3: .agg() with Dictionary and Named Aggregations (30 min)
- Show per-column agg dict: `df.groupby("region").agg({"revenue": "sum", "quantity": "mean"})`
- Show named aggregations (pandas >= 0.25): `df.groupby("region").agg(total=("revenue","sum"), avg=("revenue","mean"))`
- Show list of functions: `df.groupby("region")["revenue"].agg(["sum","mean","count"])`
- Runnable example: build a "region summary" table with total_revenue, avg_order, order_count columns

Part 4: Hands-on Practice (30 min)
- Exercise 1: Given employee DataFrame (dept, role, salary, years), compute avg salary by dept, headcount by dept, max salary by role
- Exercise 2: Given orders DataFrame (date, customer_id, product, quantity, price), compute monthly revenue, top 3 customers by total spend, revenue by product category

Key Takeaways: split-apply-combine, .groupby("col"), .agg({"col": "func"}), named agg syntax

Learning Objectives:
- Create pivot tables with pd.pivot_table()
- Use crosstab for frequency tables
- Understand index, columns, values, aggfunc parameters
- Add margins (subtotals) to pivot tables

Part 1: pd.pivot_table() Fundamentals (30 min)
- Explain pivot table: rows = index, columns = columns, cells = aggregated values
- Show: `pd.pivot_table(df, values="revenue", index="region", columns="category", aggfunc="sum", fill_value=0)`
- Show multiple aggfuncs: `aggfunc={"revenue": "sum", "quantity": "mean"}`
- Show margins=True for row/column totals
- Runnable example: regional sales pivot with Electronics/Clothing/Food categories

Part 2: pd.crosstab() (30 min)
- Explain crosstab: frequency count between two categorical columns
- Show: `pd.crosstab(df["region"], df["product"])`
- Show with values and aggfunc: `pd.crosstab(df["region"], df["product"], values=df["revenue"], aggfunc="sum")`
- Show normalize=True for percentages
- Runnable example: crosstab of customer segment vs product category (count and %)

Part 3: pd.pivot() (30 min)
- Distinguish pivot (no aggregation, requires unique index+column combos) from pivot_table
- Show: `df.pivot(index="date", columns="metric", values="value")` for wide-format time series
- Show when to use pivot vs pivot_table
- Runnable example: reshape a long-format metrics table to wide format (one column per metric)

Course header: Data Transformation | 2 hours | Intermediate

Learning Objectives:
- Convert wide format to long format with pd.melt()
- Convert long format to wide with pd.pivot() or unstack()
- Use stack() and unstack() with MultiIndex DataFrames
- Understand when each format is appropriate

Part 1: Wide vs Long Format (30 min)
- Explain the concept: wide = one row per entity, many value columns; long = one row per entity+metric pair
- When to use each: wide for display/pivot, long for groupby/plot/tidydata standards
- Show: `pd.melt(df, id_vars=["date","region"], value_vars=["q1","q2","q3","q4"], var_name="quarter", value_name="revenue")`

- Show all melt parameters: id_vars, value_vars, var_name, value_name, ignore_index

Part 3: stack() and unstack() (30 min)

- Explain unstack(): moves row level to column level (inverse of stack)
- Show with MultiIndex DataFrame from a groupby result
- Show: `result.unstack("region")` to make regions into columns
- Runnable example: groupby region+quarter result → unstack quarter to columns

Part 4: Hands-on Practice (30 min)

- Exercise 2: MultiIndex sales result (region × product) → unstack product to columns → compute % share per region per product

**lesson-05-merge-join-basics.md** — "Merging DataFrames: SQL-Style Joins"
Course header: Data Transformation | 2 hours | Intermediate

Learning Objectives:
- Understand inner, left, right, and outer joins
- Use pd.merge() to combine DataFrames on a common key
- Handle different key column names with left_on / right_on
- Detect and handle merge mismatches with indicator=True

Part 1: Join Types Explained (30 min)
- Explain each join type with a diagram in comments:
  inner: only rows where key exists in BOTH tables
  left: all rows from left, matched rows from right (NaN if no match)
  right: all rows from right, matched rows from left
  outer: all rows from both, NaN where no match
- Show: `pd.merge(orders, customers, on="customer_id", how="inner")`
- Show left join: `pd.merge(orders, customers, on="customer_id", how="left")`
- Runnable example: orders + customers tables, compare inner vs left join row counts

Part 2: Key Handling (30 min)
- Show left_on/right_on for different key names: `pd.merge(orders, customers, left_on="cust_id", right_on="id")`
- Show merging on multiple keys: `pd.merge(df1, df2, on=["year","month"])`
- Show merging on index: `df1.merge(df2, left_index=True, right_on="id")`
- Show validate parameter: validate="1:1", "1:m", "m:1", "m:m"
- Runnable example: sales + products (different key names), enrich with product name and category

Part 3: Diagnosing Merge Issues (30 min)
- Show indicator=True: adds "_merge" column ("left_only", "right_only", "both")
- Show how to find unmatched rows: `merged[merged["_merge"] == "left_only"]`
- Show suffixes for overlapping column names: `suffixes=("_orders","_products")`
- Show how duplicate keys cause row explosion (many-to-many pitfall)
- Runnable example: detect which orders have no matching customer record

Part 4: Hands-on Practice (30 min)
- Exercise 1: Enrich a transactions table with product details and store details using two sequential merges. Calculate revenue = quantity × unit_price after merge.
- Exercise 2: Find all products that have never been ordered (anti-join pattern using left merge + filter where _merge == "left_only")

Key Takeaways: merge(df1, df2, on, how), how="inner/left/right/outer", left_on/right_on for name mismatches, indicator=True for diagnostics

**lesson-09-apply-map-transform.md** — "Apply, Map, and Vectorized Operations"
Course header: Data Transformation | 2 hours | Intermediate

Learning Objectives:
- Apply element-wise functions with .map() and .apply()
- Apply row-wise or column-wise functions with DataFrame.apply()
- Prefer vectorized operations over apply() for performance
- Use np.where() and np.select() for conditional column creation

Part 1: Series.map() and Series.apply() (30 min)
- Explain .map(): element-wise transformation using a function or dict lookup
- Show: `df["grade"] = df["score"].map({90: "A", 80: "B", ...})` — dict mapping
- Show: `df["grade_letter"] = df["score"].map(lambda x: "A" if x >= 90 else "B" if x >= 80 else "C")`
- Explain .apply() on Series: same as map but supports more complex functions
- Distinguish: map for element-wise simple transforms, apply for complex single-series logic
- Runnable example: map product_id to product_name using a lookup dict

Part 2: DataFrame.apply() (30 min)
- Show axis=0 (column-wise): `df.apply(lambda col: col.max() - col.min())`
- Show axis=1 (row-wise): `df.apply(lambda row: row["price"] * row["quantity"], axis=1)`
- Show returning a Series from apply() to create multiple new columns
- Performance warning: apply(axis=1) is slow — prefer vectorized when possible
- Runnable example: row-wise revenue calculation (price × quantity × (1 - discount))

Part 3: Vectorized Alternatives (30 min)
- Show why vectorized > apply: show timing comment (apply is Python loop, vectorized is C)
- Show np.where(): `np.where(condition, true_value, false_value)` — single condition branch
