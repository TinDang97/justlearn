<objective>
Install @mlc-ai/web-llm, create the LLM worker file, implement the module-level singleton useAIEngine hook (mirroring usePyodideWorker), and build the download progress bar component.

Purpose: This is the core AI engine infrastructure. The singleton pattern ensures the WebLLM engine survives route navigation without re-initialization. The worker file isolates GPU compute from the UI thread. The progress bar provides UX during the 2-8 minute first model download. Lazy loading ensures Pyodide and WebLLM never compete for memory on mount.
Output: Working useAIEngine hook with singleton lifecycle, llm.worker.mjs, AIEngineProgress component, comprehensive tests.
</objective>

  <name>Task 1: Install @mlc-ai/web-llm + create llm.worker.mjs</name>
  <files>package.json, public/workers/llm.worker.mjs</files>
  <action>
    1. Run `pnpm add @mlc-ai/web-llm` to install the WebGPU inference engine.

    2. Create `public/workers/llm.worker.mjs` mirroring the structure of `public/workers/pyodide.worker.mjs`:
       ```javascript
       import { WebWorkerMLCEngineHandler } from '@mlc-ai/web-llm'

       const handler = new WebWorkerMLCEngineHandler()
       self.onmessage = (msg) => { handler.onmessage(msg) }
       ```
       This is the entire file. The handler receives postMessage from the CreateWebWorkerMLCEngine proxy and dispatches to the MLC engine running on the GPU. All GPU compute is isolated in this worker.

    NOTE: The worker file uses a bare import from '@mlc-ai/web-llm'. Since this is in public/ and served as a static file, the browser needs to resolve this. WebLLM's documentation shows this pattern works because CreateWebWorkerMLCEngine handles the module resolution by passing the worker URL to the Worker constructor. The import is resolved at the bundler level since Next.js serves public/ files as-is. If the bare import does not work at runtime, the fallback is to use a CDN URL: `import { WebWorkerMLCEngineHandler } from 'https://esm.run/@mlc-ai/web-llm'`. Test this during implementation — if the bare import fails in the browser, switch to the CDN URL.
  </action>

  </verify>

       - 'use client' directive
       - Export types: `AIEngineStatus = 'idle' | 'loading' | 'ready' | 'error' | 'unsupported'` and `DownloadProgress = { progress: number; text: string }`
       - Module-level variables: `let engineInstance: MLCEngineInterface | null = null` and `let enginePromise: Promise<MLCEngineInterface> | null = null`
       - Export `isWebGPUSupported()` function: checks `typeof navigator !== 'undefined' && 'gpu' in navigator && navigator.gpu !== null`
       - Export `useAIEngine(modelId: string)` hook:
         - `useState<AIEngineStatus>` initialized to 'unsupported' if !isWebGPUSupported(), else 'idle'
         - `useState<DownloadProgress | null>(null)` for progress tracking
         - `useRef` for statusRef (avoids stale closure, same pattern as usePyodideWorker)
         - `getEngine` callback (useCallback with [modelId] dep):
           - If !isWebGPUSupported(), throw Error('WebGPU not supported')
           - If engineInstance exists, return it immediately
           - If enginePromise exists, return it (dedup concurrent calls)
           - Set status to 'loading'
           - Dynamic import: `const { CreateWebWorkerMLCEngine } = await import('@mlc-ai/web-llm')`
           - Create enginePromise = CreateWebWorkerMLCEngine with:

             - modelId
             - `{ initProgressCallback: (report) => { setDownloadProgress({ progress: report.progress, text: report.text }); if (report.progress === 1) statusRef.current('ready') } }`
           - .then: set engineInstance, return engine
           - .catch: reset enginePromise to null, set status to 'error', rethrow
         - Return { getEngine, status, downloadProgress }

    CRITICAL anti-patterns to avoid (from RESEARCH.md):
    - Do NOT import @mlc-ai/web-llm at module top level — crashes SSG build with 'navigator is not defined'
    - Do NOT initialize engine in useEffect — destroys on route navigation
    - Do NOT call getEngine() on mount — 2.2GB download must be user-triggered only
