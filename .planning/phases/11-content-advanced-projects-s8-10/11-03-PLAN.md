- Problem: A CSV export from a legacy sales system has mixed dtypes, inconsistent date formats, duplicate order IDs, and nulls in critical fields. Build a pipeline that ingests it, reports quality issues, cleans the data, validates the result against a pandera schema, and prints a cleaning summary.
- Architecture: `load('data/transactions.csv')` → `profile(df)` → `clean(df)` → `validate(df, schema)` → `summarize(original_df, clean_df)` → print summary

- Step 4 (Summarize): Print: rows in → rows out, duplicates removed, nulls filled, validation errors remaining. Use string formatting for alignment.

- Problem: Build a benchmark harness that compares 3 implementations of the same pipeline: (1) naive loop-based, (2) pandas apply-based, (3) fully vectorized. Measure and report timing, memory usage, and output correctness for each variant on datasets of 3 sizes (1K, 10K, 100K rows).
- Architecture: `generate_dataset(n)` → `run_naive(df)` / `run_apply(df)` / `run_vectorized(df)` → `benchmark_all(sizes)` → `format_benchmark_report(results)` → print

- Step 2 (benchmark_all implementation): For each size in [1000, 10000, 100000], generate dataset, run each variant with `benchmark(fn, df, n=3)`, collect timing. Verify output equality with `assert_frame_equal`. Record memory with `df.memory_usage(deep=True).sum()`.
