# Course 1: Introduction to Data Engineering

**Level:** Intermediate
**Duration:** 8 lessons x 35–45 minutes = ~5 hours total
**Prerequisites:** Python fundamentals (variables, functions, loops, file I/O)

---

## Course Overview

This section establishes the mental model and vocabulary that all subsequent hands-on sections build on. Before writing a single line of pandas code, you need to understand what data engineering is, why it exists, and where it fits in the modern data ecosystem.

Data engineering is one of the fastest-growing engineering disciplines. Every company that makes decisions based on data — which is nearly every company — needs data engineers to build and maintain the systems that make that data usable. This section explains the full landscape: what data engineers do, the tools they use, the teams they work alongside, and the architectural patterns that underpin the entire discipline.

By the end of this section you will have a solid conceptual foundation for everything that follows. Sections 2–10 build hands-on skills on top of this foundation.

## What You Will Learn

By the end of this section, you will be able to:

- Define data engineering and explain its purpose in a business context
- Describe the four layers of the modern data stack and name tools at each layer
- Identify Python's role in data engineering and the key libraries you will use throughout this course
- Explain the anatomy of a data pipeline and distinguish ETL from ELT
- Compare batch and streaming processing and select the appropriate paradigm for a use case
- Describe the main storage types (OLTP database, data warehouse, data lake, lakehouse) and when each is appropriate
- List the 6 dimensions of data quality and explain where quality failures commonly originate
- Distinguish between data engineer, analytics engineer, data analyst, data scientist, and MLOps engineer roles
